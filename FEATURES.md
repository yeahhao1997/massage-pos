# 功能清单 / Feature List

> 马来西亚按摩院多商户平台（参考 Fresha）。会员卡全平台通用 + 店间结算 + 顾客端在线预约。
> Malaysia multi-shop massage platform. Universal member card · inter-shop settlement · online booking.

最后更新：2026-06-06

---

## 一、平台管理员 Platform Admin（账号 admin）

| 功能 | 说明 | 状态 |
|---|---|---|
| 平台概览 Overview | 入驻门店数、平台会员数、钱包总余额(负债)、期间营业额/订单/充值 | ✅ |
| 门店入驻 Onboard Shop | 新增合作门店，自动创建该店老板登录账号 | ✅ |
| 门店管理 | 查看各店技师/订单概况，暂停/恢复营业 | ✅ |
| **店间结算 Settlement** | 每店：收的充值款 − 核销的服务额 = 与平台净额（谁欠谁一目了然） | ✅ |

## 二、门店柜台 Shop Counter（账号 shop_owner / shop_staff）

| 模块 | 功能 | 状态 |
|---|---|---|
| 🏠 工作台 | 今日营业额/充值/新会员/预约一览 | ✅ |
| 💳 会员卡 | 开卡、储值充值(含赠送+积分)、余额扣款、次卡核销、消费流水 | ✅ |
| 💳 会员卡（通用） | **会员全平台通用**：任意合作店都能查到、充值、消费同一张卡 | ✅ |
| 📅 预约排班 | 按日查看、技师时间冲突检测、到店→完成→取消状态流转 | ✅ |
| 🧾 收银台 | 服务+商品开单、按技师算提成、余额/现金/TnG/GrabPay/DuitNow/FPX 多支付 | ✅ |
| 📊 报表 | 营业额、支付方式分布、技师业绩提成、近30天趋势 | ✅ |
| ⚙️ 设置 | 员工技师、服务项目、商品、房间、卡类型维护 | ✅ |
| 🔒 数据隔离 | 每家店只看自己的技师/服务/订单/报表 | ✅ |

## 三、顾客端 Customer App（网址 /m，免登录浏览）

| 功能 | 说明 | 状态 |
|---|---|---|
| 找店 Find Shops | 按城市筛选门店，看简介、起步价、服务数 | ✅ |
| 门店详情 | 服务列表、价格、时长、技师 | ✅ |
| 在线预约 | 选服务/日期/时间/技师 → 提交，自动算结束时间，柜台同步收到 | ✅ |
| 会员登录 | 卡号 + 手机号登录 | ✅ |
| **我的钱包** | 跨店通用余额、积分、次卡、**跨店消费记录**、我的预约 | ✅ |

## 四、技术 & 基础能力 Platform Foundation

| 能力 | 说明 |
|---|---|
| 多商户架构 | 业务表按 `tenant_id` 隔离；会员与钱包在平台层共享 |
| 登录权限 | 三种角色：平台管理员 / 店老板 / 店员；密码 scrypt 加密；令牌会话 |
| 中央钱包 | 会员储值是平台对会员的负债，每笔记录发生门店，支撑店间清结算 |
| 本地化 | 货币 RM(MYR)，马来西亚支付方式，中英双语一键切换 |
| 数据 | 单文件 SQLite (`data/platform.db`)，拷走即完整备份 |
